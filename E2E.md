# E2E проверки (Playwright)

В проекте настроены smoke e2e-тесты для базовой браузерной проверки UI.

## Что проверяется
- Открытие главной страницы `/`
- Переход в `/tools/rice`
- Добавление новой фичи в бэклог
- Прогон в двух режимах: desktop (`chromium`) и mobile (`Pixel 7`)

## Запуск
```bash
npm run test:e2e
```

Интерактивный режим:
```bash
npm run test:e2e:ui
```

## Когда запускать
- Обязательно перед крупными UI-изменениями и после них
- Желательно перед merge в `main`

## Важно
- E2E — отдельный (optional) шаг, не блокирует обычный локальный цикл разработки
- Браузерные артефакты (`playwright-report`, `test-results`, `blob-report`) игнорируются через `.gitignore`
