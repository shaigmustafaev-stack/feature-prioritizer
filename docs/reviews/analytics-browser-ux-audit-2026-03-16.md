# Analytics Browser UX Audit

Date: 2026-03-16
Scope: Browser-only audit of analytics feature (`/tools/analytics`, `/tools/analytics/new`, mobile viewport 390)
Mode: Review only, no code changes

## What was tested in browser
- Desktop flow: empty state, create dashboard, add metric/periods, edit period labels, switch Data/Dashboard tabs, attempt AI analysis, auth gate behavior.
- Mobile flow (390w): same path + tap-target sizes + horizontal scroll + numeric input behavior.
- Share URL behavior check: `/share/test`.

## Critical findings

### 1) Numeric input resets to `0` after blur when user clears value
- Reproduced in browser repeatedly:
  - during edit: `""`
  - after blur/wait: `"0"`
- User impact: hard to edit numbers naturally, creates “input is fighting me” feeling.
- File refs:
  - `app/components/MetricInput.tsx:73`
  - `app/components/MetricInput.tsx:77`

### 2) Mobile tap-targets are below 44px for core actions
- Measured in mobile browser run (390w):
  - Save: 32px
  - Data/Dashboard tabs: 23px
  - Add metric: 32px
  - Add period: 32px
  - Upload CSV/XLSX: 32px
- User impact: missed taps, slow operation on phone.
- File refs:
  - `app/tools/analytics/[id]/page.tsx:140`
  - `app/tools/analytics/[id]/page.tsx:179`
  - `app/tools/analytics/[id]/page.tsx:182`
  - `app/components/CsvUploadButton.tsx:84`
  - `app/components/MetricInput.tsx:119`

### 3) Guest can do all input work, block appears only at final action
- Current behavior: user spends time filling dashboard, then sees auth gate on Analyze/Save/Share.
- User impact: frustration and perceived data loss risk.
- File refs:
  - `app/hooks/useAnalytics.ts:82`
  - `app/hooks/useAnalytics.ts:333`
  - `app/hooks/useAnalytics.ts:394`
  - `app/hooks/useAnalytics.ts:419`

## High-priority product gaps (PM -> leadership use case)

### 4) AI insights are not editable by PM
- Need: PM should be able to rewrite insight text before sharing upward.
- Current: read-only rendering.
- File refs:
  - `app/components/ChartBlock.tsx:45`

### 5) No “show only chart” presentation mode
- Need: hide insight block to present clean visual dashboard.
- Current: only detail expand/collapse; summary stays.
- File refs:
  - `app/components/ChartBlock.tsx:47`

### 6) Data and charts are split by tabs (no single cockpit view)
- Need: one-screen decision surface for PM review.
- Current: forced tab-switch context.
- File refs:
  - `app/tools/analytics/[id]/page.tsx:134`

### 7) Share-link governance is weak
- No revoke/expiry/password.
- `share_id` generated by `Math.random`.
- File refs:
  - `app/api/analytics/share/route.ts:40`

## Medium findings

### 8) Auto-save errors are silent
- Current: catch block suppresses failures.
- Impact: users think data is saved while it may not be.
- File refs:
  - `app/hooks/useAnalytics.ts:148`

### 9) Analyze button is available without quality checks on data completeness
- Current validation is minimal; weak input can go to analysis.
- File refs:
  - `app/tools/analytics/[id]/page.tsx:189`

### 10) Dark theme readability feels too muted
- Visual signal is low for dashboard-heavy page.
- Impact: slower scanning, lower executive readability.

### 11) Invalid share page lacks friendly explanation
- `/share/test` returns technical 404 shell.
- Better: explain link expired/invalid and where to request new one.

## Confirmed good
- Period labels are editable.
- CSV/XLSX upload is available.
- No horizontal scroll on tested mobile viewport.
- Chart fits mobile width and renders correctly.
- Auth-gate text is clear.

## Priority fix order (implementation backlog)
1. Fix numeric input reset-to-zero behavior.
2. Raise all core mobile controls to min 44px tap-target.
3. Show early auth requirement state for guest mode in editor.
4. Add presentation mode: “charts only / hide AI”.
5. Add manual editing of AI insights (PM override layer).
6. Add export for leadership sharing (PDF/PNG first, then richer formats).
7. Add share-link controls (revoke, expiry).
