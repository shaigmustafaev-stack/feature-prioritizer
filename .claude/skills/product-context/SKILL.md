# ProductHub — Product Context

Этот скил даёт контекст о продукте, архитектуре и паттернах. Читай нужные docs/ файлы в зависимости от задачи.

## Когда какой файл читать

| Задача | Файл |
|--------|------|
| Новый инструмент, новая страница, связь между инструментами | `docs/prd-summary.md` + `docs/tool-patterns.md` |
| Работа с Feature, Supabase, API routes, RLS, миграция данных | `docs/data-model.md` |
| Создание хука, компонента, страницы по существующему паттерну | `docs/tool-patterns.md` |
| Понять что готово / что нет, приоритеты релизов, аудиторию | `docs/prd-summary.md` |

## Быстрый контекст

- **Стек:** Next.js 16, React 19, shadcn/ui (base-ui), Tailwind 4, Supabase, Vercel
- **Дизайн-система:** описана в секции "Дизайн-система (правила)" в `CLAUDE.md` — не дублировать
- **Центральная сущность:** Feature — рождается из идеи/фидбека, получает оценку, движется по roadmap
- **Два режима:** анонимный (localStorage) и авторизованный (Supabase через `/api/features`)
- **MVP готов:** Приоритизатор фич (RICE/ICE) + авторизация. Следующий: Claude API + Генератор фич
