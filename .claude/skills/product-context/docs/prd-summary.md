# PRD Summary

## Проблема
PM работают в разрозненных инструментах. Фичи, метрики и фидбек не связаны. ProductHub — единый workspace, объединяющий ключевые инструменты и усиливающий их AI.

## Аудитория
PM / Senior PM / Product Owner / Product Analyst в продуктовых командах. Первый этап — Битрикс24.

## 5 инструментов

| # | Инструмент | Статус | Путь | AI |
|---|-----------|--------|------|-----|
| 1 | Приоритизатор фич | **ГОТОВ** | `/tools/rice` | Claude предлагает оценки по описанию |
| 2 | Генератор фич и описаний | планируется | `/tools/generator` | Ядро — Claude генерирует User Stories, AC |
| 3 | Roadmap продукта | планируется | `/tools/roadmap` | Claude предлагает колонку по готовности |
| 4 | Аналитика продукта | **ГОТОВ** | `/tools/analytics` | OpenRouter AI анализирует метрики, даёт рекомендации |
| 5 | Сбор фидбека и багов | планируется | `/tools/feedback` | Claude категоризирует и находит паттерны |

## Связи между инструментами (flow)

```
Фидбек → "Создать фичу" → Приоритизатор → "В roadmap" → Roadmap
                                ↓
                          Генератор фич (описание + User Stories)
                                ↓
                           Аналитика (данные для обоснования)
```

Feature — центральная сущность. Рождается из фидбека/идеи → описание в генераторе → оценка в приоритизаторе → движется по roadmap.
